## [ch4]: components and props

    - 根據需求切割元件，元件有時候不是切得越細越好，找出有重複性或獨特唯一性來做出元件

    - 父元件主要放置資料，把"資料"或"函式"傳遞到子元件，子元件可以修改資料，但是是透過父元件函式來修改．
    最終修改後改變的是父層資料(父元件一改動資料，子元件因為有偵測到props改動，所以重新執行 render)


## flow 圖示

``

            data or function
       ------------------------> 
    父                             子
       <------------------------
            return update data

``
